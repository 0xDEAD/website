{{/* This is fairly contrived: - Page.Next uses publish date, which is not what we want, we want by weight. - Pages.Prev and Pages.Next will wrap around instead of returning nil on the first/last item (!!!!) - eq directly on Page doesn't work, so compare their link instead. - Page.CurrentSection returns the actual current section, but for news we want the root one so use FirstSection, which was added in Hugo 0.45.1. So we first need to find our section, then reorder, then take prev/next based on this temporary list, then manually figure out if wraparound occured via comparison on the page URL. Refs: - https://pkg.go.dev/github.com/spf13/hugo/hugolib#Position - https://github.com/gohugoio/hugo/blob/master/hugolib/pagesPrevNext.go */}} {{/* This enables grabbing all pages, including descendants in subsections. This is needed for /news/. It'd be nice to find a better way to achieve this without a silly loop, like it's done in shortcodes/cutetable.html. */}} {{ $page := .Page }} {{ range $section := $page.Site.Sections }} {{ if hasPrefix $page.RelPermalink $section.RelPermalink }} {{ $pages := (where .Site.AllRegularPages "Section" $section.Section).ByWeight.Reverse }} {{ $prev := $pages.Prev $page }} {{ $next := $pages.Next $page }} {{ $first := index $pages 0 }} {{ $indexLast := sub $pages.Len 1 }} {{ $last := index $pages $indexLast }} {{/* In Go 1.11, it is possible to update a variable but not in previous Go versions, so we need to create more temporary variable to get to the exact values we want. */}} {{ $isFirst := not $prev}} {{ $isLast := not $next }} {{ $hasPrev := not $isFirst }} {{ $hasNext := not $isLast }} {{ if or $hasPrev $hasNext }}

{{ if $hasPrev }} ← {{$prev.Title}} {{ end }} {{ if and $hasPrev $hasNext }} | {{ end }} {{ if $hasNext }} {{$next.Title}} → {{ end }}

{{ end }}
